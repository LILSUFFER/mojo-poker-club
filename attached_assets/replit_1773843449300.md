### Docker Network
- All containers are on a single Docker network: `bot1_pet`
- This is required because the `nginx` container (defined in bot1's docker-compose) needs to proxy requests to both bot1 (`pet:9000`) and bot2 (`pet2:9000`, `node_server2:3000`)
- **IMPORTANT**: Bot2's docker-compose services MUST use unique service names (`pgdb2`, `redis2`) to avoid Docker DNS conflicts. Previously they were named `pgdb` and `redis` which caused DNS to resolve to wrong containers.

### Nginx Configuration
- Nginx runs in a container defined in `/opt/bot1/docker-compose.yml`
- Config files are at `/opt/bot1/docker/nginx/`
  - `default.conf` — handles `support.nukapay.com` → proxies to `app:9000`
  - `playwallet.conf` — handles `support.playwallet.bot` → proxies to `pet2:9000`
- **Dynamic DNS resolution**: Both configs use `resolver 127.0.0.11 valid=5s` with `set $upstream` variables instead of hardcoded hostnames in `fastcgi_pass`/`proxy_pass`. This ensures nginx re-resolves DNS every 5 seconds, so container restarts don't break routing.
- SSL certificates are from Let's Encrypt at `/etc/letsencrypt/live/`

## Issues Fixed (History)

1. **Bot1 Redis port** — `.env` had `REDIS_PORT=6380:6379` (Docker mapping syntax), fixed to `6379`; docker-compose hardcodes `6380:6379` port mapping
2. **CORS for ru.playwallet.bot** — added to ALLOWED_ORIGINS in bot2 `.env`
3. **Nginx no-cache headers** — added for `/live_chat/` location block
4. **Widget cache busting** — dynamic `Date.now()` in widget.js loader
5. **Mobile widget keyboard** — overlay approach for iOS Safari (ongoing)
6. **`isBanned() on null` crash** — added null check for `$this->botUser` in `TelegramBotController.php` (both bots)
7. **Docker DNS conflict** — Bot2 services renamed from `pgdb`/`redis` to `pgdb2`/`redis2` to prevent DNS collision in shared network
8. **Nginx static DNS** — added `resolver 127.0.0.11 valid=5s` and `set $upstream` variables for dynamic resolution
9. **Bot2 branding** — replaced NukaPay logo/text with PlayWallet logo/text
10. **Unique TELEGRAM_SECRET_KEY** — each bot now has its own webhook secret
11. **Container restart policies** — all containers set to `restart: always`
12. **Removed /start deleteMessage** — `/start` command now stays visible in chat
13. **bootstrap/cache directory** — fixed permissions after cache clear broke Laravel

## Important Warnings

- **Never use same Docker service names** across bot1 and bot2 docker-compose files (they share the `bot1_pet` network)
- **After `docker compose up -d` for bot2** — nginx may need reload: `docker exec nginx nginx -s reload`
- **After editing `.env`** — must restart containers AND clear cache: `docker exec petX php artisan optimize:clear`
- **Bot2 code is a copy of Bot1** — any app-level changes need to be made in both `/opt/bot1` and `/opt/bot2` separately
- **Redis ports**: Bot1 redis exposed on host port 6380 (mapped to container 6379), Bot2 redis not exposed on host
